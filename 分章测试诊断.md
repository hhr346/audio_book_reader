## 🐛 发现的问题

### 问题 1: 《现实一种》总页数不一致

**症状**:
- 显示的总页数 ≠ 翻到最后一页显示的页数
- 例如：显示总页数 50，但最后一页显示 45/50

**可能原因**:
1. `_totalPages` 是预计算的（逐章累加）
2. 但实际分页时，每章的实际页数可能与预计算不同
3. 因为 `getChapterPageCount()` 和实际 `_paginateText()` 可能产生细微差异

**根本原因**:
```dart
// 预计算总页数
_totalPages = Σ(getChapterPageCount(i))

// 但实际加载时
pages = _paginateText(text)
actualPageCount = pages.length

// 这两个可能不一致！
```

---

### 问题 2: 《现实一种》第二章第一页向前翻到第一章第一页

**症状**:
- 在第二章第 1 页点击"上一页"
- 预期：跳转到第一章最后一页
- 实际：跳转到第一章第 1 页

**问题代码**:
```dart
Future<void> _previousChapter() async {
  if (_currentChapterIndex > 0) {
    final prevChapterIndex = _currentChapterIndex - 1;
    
    // 获取上一章的页数（从缓存或重新计算）
    final prevChapterPageCount = _chapterPageCounts[prevChapterIndex] ?? 
        await EpubService().getChapterPageCount(...);
    
    final lastPageIndex = prevChapterPageCount - 1;  // ← 这里
    
    // 但加载后实际页数可能不同！
    await _loadCurrentChapter(forceFirstPage: false);
  }
}
```

**问题**:
- `prevChapterPageCount` 是预计算的
- 但 `_loadCurrentChapter()` 中实际分页可能产生不同页数
- 导致 `lastPageIndex` 可能超出实际范围

---

### 问题 3: 《白老虎》一直在第一章

**症状**:
- 翻到第一章最后一页
- 点击"下一页"
- 又回到第一章第 1 页（无法进入第二章）

**可能原因**:
1. `_nextChapter()` 条件判断失败
2. 章节加载失败但无错误提示
3. `widget.book.totalChapters` 不正确

**检查点**:
```dart
Future<void> _nextChapter() async {
  if (_currentChapterIndex < widget.book.totalChapters - 1) {
    // 如果 totalChapters 不正确，这里可能进不去
  }
}
```

---

## 🔧 解决方案

### 修复 1: 使用实际分页页数

**核心思想**: 不使用预计算的页数，使用实际分页后的 `pages.length`

**修改**:
```dart
// _previousChapter 中
Future<void> _previousChapter() async {
  if (_currentChapterIndex > 0) {
    final prevChapterIndex = _currentChapterIndex - 1;
    
    // 先加载上一章
    setState(() => _currentChapterIndex = prevChapterIndex);
    await _loadCurrentChapter(forceFirstPage: false);
    
    // 使用实际分页后的页数
    final lastPageIndex = _pages.length - 1;
    
    setState(() {
      _currentPageIndex = lastPageIndex;
    });
    
    // 重新计算累计页数
    _cumulativePagesRead = await _calculateCumulativePages(prevChapterIndex, lastPageIndex);
    await _updateProgress();
  }
}
```

---

### 修复 2: 总页数使用实际累加

**核心思想**: 在加载过程中动态更新总页数

**修改**:
```dart
// 在 _loadCurrentChapter 中
final pages = _paginateText(text);

// 更新该章的实际页数
_chapterPageCounts[_currentChapterIndex] = pages.length;

// 重新计算总页数
_totalPages = 0;
for (int i = 0; i < widget.book.totalChapters; i++) {
  _totalPages += _chapterPageCounts[i] ?? 0;
}
```
