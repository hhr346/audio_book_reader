# 🔊 后台播放功能配置说明

**日期**: 2026-03-13  
**状态**: ✅ 已配置  
**功能**: 锁屏/切换应用后 TTS 继续播放

## ✅ 已完成的配置

### 1. iOS Info.plist 配置

**文件**: `ios/Runner/Info.plist`

**添加内容**:
```xml
<key>UIBackgroundModes</key>
<array>
  <string>audio</string>
</array>
```

**作用**: 告诉 iOS 系统这是一个音频应用，允许后台播放

---

### 2. TTS 音频类别配置

**文件**: `lib/services/tts_service.dart`

**修改内容**:
```dart
await _flutterTts.setIosAudioCategory(
  IosTextToSpeechAudioCategory.playback,  // 播放模式（支持后台）
  [
    IosTextToSpeechAudioCategoryOptions.mixWithOthers,  // 与其他音频混合
    IosTextToSpeechAudioCategoryOptions.duckOthers,     // 降低其他音频音量
  ],
);
```

**参数说明**:
- `playback`: 播放模式，支持后台播放
- `mixWithOthers`: 允许与其他音频同时播放
- `duckOthers`: 当 TTS 播放时，自动降低其他音频的音量

## 📱 iOS 后台音频机制

### 后台模式

**UIBackgroundModes - audio**:
- 允许应用在后台继续播放音频
- 系统不会 suspend 该应用
- 电池消耗会增加

### 音频类别

**IosTextToSpeechAudioCategory.playback**:
- 用于媒体播放
- 支持后台播放
- 不受静音开关影响
- 与其他音频混合播放

**选项**:
- `mixWithOthers`: 与其他音频同时播放
- `duckOthers`: TTS 播放时降低其他音频音量

## 🔧 故障排查

### 问题 1: 锁屏后停止播放

**可能原因**:
- Info.plist 未配置 `UIBackgroundModes`
- 音频类别设置错误

**解决方法**:
```bash
# 检查 Info.plist
cat ios/Runner/Info.plist | grep -A 2 UIBackgroundModes

# 应该看到:
# <key>UIBackgroundModes</key>
# <array>
#   <string>audio</string>
# </array>
```

---

### 问题 2: 切换应用后停止播放

**可能原因**:
- 音频类别不是 `playback` 模式
- 系统资源限制

**解决方法**:
1. 确认 `setIosAudioCategory` 使用 `playback`
2. 检查控制台日志
3. 重启应用测试

---

### 问题 3: 电池消耗过快

**原因**:
- 后台播放会持续消耗电量
- TTS 合成需要 CPU

**优化建议**:
- 使用定时关闭功能
- 不使用时停止播放
- 降低屏幕亮度

## ⚠️ 注意事项

### 1. 电池消耗

**后台播放会持续消耗电量**

**建议**:
- 不使用时停止播放
- 使用定时关闭功能
- 注意电量

---

### 2. 系统限制

**iOS 可能限制后台运行时间**

**情况**:
- 长时间后台运行可能被系统 suspend
- 内存不足时可能被终止

**解决**:
- 定期回到前台应用
- 关闭不用的应用释放内存

---

### 3. 耳机控制

**线控耳机可能不支持切歌**

**原因**:
- flutter_tts 只支持基本播放
- 不支持媒体控制中心

**未来优化**:
- 添加音频播放器包装
- 支持锁屏控制
