## 🔴 修复的问题

### 问题 1: 总页数计算不一致

**症状**:
- 总页数显示 50 页
- 但各章节页数相加等于 55 页
- 进度计算不准确

**根本原因**:
两种不同的计算方法导致差异：

```dart
// 方法 1: 全文一次性计算（旧）
_totalPages = (全书总字符数) ÷ (每页容量)

// 方法 2: 逐章计算后累加（新）
_totalPages = Σ(每章字符数 ÷ 每页容量)
```

**数学不等价**！

**例子**:
```
第 1 章：105 字 ÷ 50 字/页 = 2.1 → 3 页（向上取整）
第 2 章：105 字 ÷ 50 字/页 = 2.1 → 3 页（向上取整）
第 3 章：105 字 ÷ 50 字/页 = 2.1 → 3 页（向上取整）

方法 1: 315 字 ÷ 50 字/页 = 6.3 → 7 页
方法 2: 3 + 3 + 3 = 9 页

差异：7 页 vs 9 页！
```

**修复方案**:
统一使用逐章累加方法，确保准确性。

**代码变更**:
```dart
// 新的 _calculateTotalPages()
Future<void> _calculateTotalPages() async {
  final chapters = await EpubService().getChapters(widget.book.filePath);
  
  _chapterPageCounts.clear();
  _totalPages = 0;
  
  for (int i = 0; i < chapters.length; i++) {
    final pageCount = await EpubService().getChapterPageCount(
      widget.book.filePath,
      i,
      fontSize: _fontSize,
      lineHeight: _lineHeight,
    );
    _chapterPageCounts[i] = pageCount;
    _totalPages += pageCount;
  }
  
  print('📊 全书总页数：$_totalPages (共${chapters.length}章)');
  print('📊 各章页数：${_chapterPageCounts.values.join(", ")}');
}
```

### 问题 2: 章节切换页码突变

**症状**:
- 第 1 章第 10 页 → 点击"下一页"
- UI 瞬间显示 "第 1/50 页"
- 然后突然跳变成 "第 11/50 页"
- 进度条也跟随跳变

**根本原因**:
状态更新不同步：

```dart
// 旧代码
_nextChapter() {
  setState(() {
    _currentChapterIndex++;
    _currentPageIndex = 0;  // ← 先设置为 0
  });
  // UI 立即刷新，显示"第 1 页"
  
  await _loadCurrentChapter(); 
  // ↓ 这里才计算累计页数
  _cumulativePagesRead = await _calculateCumulativePages(...);
  
  // 进度条突然跳变
}
```

**时间轴**:
1. `setState(() { _currentPageIndex = 0; })` → UI 刷新
2. 用户看到 "第 1 页"
3. `_calculateCumulativePages()` 执行
4. `_updateProgress()` 更新进度
5. UI 再次刷新，显示 "第 11 页"

**修复方案**:
在同一个 `setState` 中更新所有相关状态。

**代码变更**:
```dart
// 新的 _nextChapter()
Future<void> _nextChapter() async {
  final nextChapterIndex = _currentChapterIndex + 1;
  
  // 提前计算累计页数
  final cumulativePages = await _calculateCumulativePages(nextChapterIndex, 0);
  
  setState(() {
    _currentChapterIndex = nextChapterIndex;
    _currentPageIndex = 0;
    _cumulativePagesRead = cumulativePages; // ← 同时更新
  });
  
  print('📖 切换到下一章：第${nextChapterIndex + 1}章，累计第${cumulativePages + 1}页');
  
  await _loadCurrentChapter(forceFirstPage: true);
}
```

**效果**:
- UI 只刷新一次
- 直接显示正确的累计页数
- 无跳变

### 问题 3: 翻页时重复计算累计页数

**症状**:
- 每次翻页都要重新计算累计页数
- 性能浪费
- 可能导致计算不一致

**修复方案**:
在当前章节内翻页时，直接 +/- 1。

**代码变更**:
```dart
// 新的 _nextPage()
Future<void> _nextPage() async {
  if (_currentPageIndex < _pages.length - 1) {
    final nextPageIndex = _currentPageIndex + 1;
    
    // 在当前章节内，只需 +1
    _cumulativePagesRead++;
    
    setState(() {
      _currentPageIndex = nextPageIndex;
    });
    
    await _updateProgress();
    
    print('📖 下一页：第${_currentChapterIndex + 1}章 第${nextPageIndex + 1}页，累计${_cumulativePagesRead + 1}/$_totalPages 页');
    
    if (_isTtsPlaying) {
      TtsService().speak(_pages[_currentPageIndex]);
    }
  } else {
    await _nextChapter();
  }
}
```

## 💡 技术细节

### 页数缓存机制

```dart
// 缓存所有章节的页数
final Map<int, int> _chapterPageCounts = {};

// 使用时先查缓存
final pageCount = _chapterPageCounts[chapterIndex] ?? 
    await EpubService().getChapterPageCount(...);

// 缓存结果
_chapterPageCounts[chapterIndex] = pageCount;
```

**优点**:
- 避免重复计算
- 提高性能
- 字体改变时清空缓存重新计算

---

### 状态同步机制

```dart
// 旧代码：分步更新
setState(() { _currentPageIndex = 0; });
// UI 刷新 → 显示"第 1 页"

_cumulativePagesRead = await _calculateCumulativePages(...);
// 计算累计页数

setState(() { /* 更新进度 */ });
// UI 再次刷新 → 显示"第 11 页"

// 新代码：同步更新
final cumulativePages = await _calculateCumulativePages(...);
// 先计算

setState(() {
  _currentChapterIndex = nextChapterIndex;
  _currentPageIndex = 0;
  _cumulativePagesRead = cumulativePages; // 同时更新
});
// UI 只刷新一次 → 直接显示正确值
```
