# 📊 页数统计问题分析

**日期**: 2026-03-12  
**问题**: 总页数不等于各章节页数总和 + 章节切换页码突变

---

## 🔍 问题根源

### 问题 1: 总页数计算不一致

**当前实现**:

```dart
// 方法 1: 全文总页数（一次性计算）
_totalPages = await EpubService().calculateTotalPages(filePath, ...);
// 计算方式：全书总字符数 ÷ 每页容量

// 方法 2: 各章节页数累加（逐章计算）
for (int i = 0; i < chapterIndex; i++) {
  cumulative += await EpubService().getChapterPageCount(i);
}
// 计算方式：每章字符数 ÷ 每页容量，然后累加
```

**问题**:
- 方法 1：`(总字符数) ÷ 每页容量 = 总页数`
- 方法 2：`Σ(每章字符数 ÷ 每页容量) = 总页数`

**数学上不等价**！

**例子**:
- 第 1 章：100 字 ÷ 50 字/页 = 2 页
- 第 2 章：100 字 ÷ 50 字/页 = 2 页
- 方法 2 总计：2 + 2 = 4 页

- 全书：200 字 ÷ 50 字/页 = 4 页
- 方法 1 总计：4 页

看起来一样？但如果：
- 第 1 章：105 字 ÷ 50 字/页 = 2.1 → 3 页（向上取整）
- 第 2 章：105 字 ÷ 50 字/页 = 2.1 → 3 页（向上取整）
- 方法 2 总计：3 + 3 = 6 页

- 全书：210 字 ÷ 50 字/页 = 4.2 → 5 页（向上取整）
- 方法 1 总计：5 页

**差异**: 6 页 vs 5 页！

### 问题 2: 章节切换页码突变

**当前流程**:

```dart
// _nextChapter()
setState(() {
  _currentChapterIndex++;
  _currentPageIndex = 0;  // ← 设置为 0
});
await _loadCurrentChapter(forceFirstPage: true);

// _loadCurrentChapter()
_cumulativePagesRead = await _calculateCumulativePages(_currentChapterIndex, 0);
// ↑ 这里才重新计算累计页数

setState(() {
  _pages = pages;
  _currentPageIndex = 0;
  _isLoading = false;
});

// 更新进度
final progress = _totalPages > 0 
    ? ((_cumulativePagesRead + 1) / _totalPages * 100).round();
```

**问题**:
1. `setState(() { _currentPageIndex = 0; })` 先执行
2. UI 立即刷新，显示 "第 1 页"
3. 然后才计算 `_cumulativePagesRead`
4. 进度条突然跳变

**用户看到的效果**:
- 第 1 章第 10 页 → 点击"下一页"
- UI 瞬间显示："第 1/5 页"（累计页数还没更新）
- 然后突然变成："第 11/50 页"（累计页数更新后）

---

## ✅ 解决方案

### 方案 1: 统一页数计算（推荐）

**核心思想**: 只使用一种计算方法 - 逐章累加

**优点**:
- 准确反映实际页数
- 考虑了每章的独立分页
- 避免取整误差累积

**实现**:
```dart
// 不再使用 calculateTotalPages()
// 改为在进入图书时计算所有章节的页数

Future<void> _calculateAllChapterPages() async {
  final chapters = await EpubService().getChapters(widget.book.filePath);
  
  _chapterPageCounts.clear();
  _totalPages = 0;
  
  for (int i = 0; i < chapters.length; i++) {
    final pageCount = await EpubService().getChapterPageCount(
      widget.book.filePath,
      i,
      fontSize: _fontSize,
      lineHeight: _lineHeight,
    );
    _chapterPageCounts[i] = pageCount;
    _totalPages += pageCount;
  }
  
  print('📊 总页数：$_totalPages (共${chapters.length}章)');
}
```

### 方案 2: 修复章节切换的页码突变

**核心思想**: 在同一个 setState 中更新所有相关状态

**实现**:
```dart
Future<void> _nextChapter() async {
  if (_currentChapterIndex < widget.book.totalChapters - 1) {
    final nextChapterIndex = _currentChapterIndex + 1;
    
    // 提前计算累计页数
    final cumulativePages = await _calculateCumulativePages(nextChapterIndex, 0);
    
    setState(() {
      _currentChapterIndex = nextChapterIndex;
      _currentPageIndex = 0;
      _cumulativePagesRead = cumulativePages;  // ← 同时更新
    });
    
    await _loadCurrentChapter(forceFirstPage: true);
  }
}
```

## 📝 修复计划

1. 移除 `calculateTotalPages()` 调用

### 第二步：修复章节切换

1. 在 `_nextChapter()` 中提前计算累计页数
2. 在同一个 setState 中更新所有状态
3. `_previousChapter()` 同样处理

### 第三步：优化缓存

1. 缓存所有章节的页数
