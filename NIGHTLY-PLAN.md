## 开发流程

### 每个任务执行步骤
1. **分析需求** - 理解功能逻辑
2. **编写代码** - 实现功能
3. **代码审查** - 检查代码质量
4. **提交 git** - 规范 commit message
5. **更新文档** - 同步开发进度

### Commit 规范
```
feat: 添加新功能
fix: 修复 bug
docs: 文档更新
style: 代码格式化
refactor: 重构
perf: 性能优化
test: 测试相关
chore: 构建/工具相关
```

### 示例
```bash
git add -A
git commit -m "feat: 添加 TTS 后台播放支持

- 配置 iOS 后台音频权限
- 实现锁屏控制
- 添加后台服务保活

Closes #1"
git push origin main
```
