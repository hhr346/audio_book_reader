```dart
Future<List<Chapter>> getChapters(String filePath) async {
  final epubBook = await parseEpub(filePath);
  final chapters = <Chapter>[];

  if (epubBook.chapters != null) {
    for (int i = 0; i < epubBook.chapters!.length; i++) {
      final chapter = epubBook.chapters![i];
      chapters.add(Chapter(
        title: chapter.title ?? '第${i + 1}章',
        content: chapter.htmlContent ?? '',
        index: i,
      ));
    }
  }

  return chapters;
}
```

## 🔧 可能的根本原因

### 原因 1: epub_plus 解析问题

**问题**: epub_plus 库可能无法正确解析某些 EPUB 结构

**白老虎可能的问题**:
- 使用了非标准的章节分隔
- 使用了嵌套的 HTML 结构
- 使用了特殊的 XML 命名空间

**解决方案**:
- 检查 epub_plus 版本
- 尝试使用其他 EPUB 解析库
- 手动解析 EPUB 文件

---

### 原因 2: EPUB 文件结构问题

**问题**: 白老虎 EPUB 可能使用了不同的结构

**检查方法**:
```bash
# 解压 EPUB（EPUB 本质是 ZIP）
cd ~/Downloads

# 查看目录结构
# 检查是否有多个章节文件
```

**预期结构**:
```

  chapter1.xhtml
  chapter2.xhtml
  chapter3.xhtml
  ...
META-INF/
  container.xml
```

**可能的异常结构**:
```
OEBPS/
  Text/
    chapter1.xhtml
    chapter2.xhtml
  ...
```

---

### 原因 3: 章节识别逻辑问题

**问题**: epub_plus 可能使用特定规则识别章节

**可能的问题**:
- 只识别有 `id` 属性的章节
- 只识别特定类的 div
- 忽略某些类型的章节

**检查代码**:
需要查看 epub_plus 的源码或文档

Future<EpubBook> parseEpub(String filePath) async {
  final file = File(filePath);
  final bytes = await file.readAsBytes();
  final epubBook = await EpubReader.readBook(bytes);
  
  print('📖 EPUB 解析结果:');
  print('  标题：${epubBook.title}');
  print('  作者：${epubBook.author}');
  print('  章节数：${epubBook.chapters?.length}');
  print('  封面：${epubBook.coverImage != null}');
  
  if (epubBook.chapters != null) {
    for (int i = 0; i < epubBook.chapters!.length; i++) {
      final chapter = epubBook.chapters![i];
      print('  第${i+1}章:');
      print('    标题：${chapter.title}');
      print('    内容长度：${chapter.htmlContent?.length}');
      print('    前 100 字符：${chapter.htmlContent?.substring(0, 100)}');
    }
  }
  
  return epubBook;
}
